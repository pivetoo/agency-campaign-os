## Onde o Kanvas já ganha

3 features que **nenhum concorrente cobre direito**:

1. **Pipeline comercial real** — todos têm "campanha", mas não pipeline com stages configuráveis, oportunidades e histórico. Você é o único.
2. **Propostas com share link rastreável** — propostas que viram "proposalView" e mostram quando o cliente abriu. Nenhum concorrente faz isso.
3. **Financeiro interno completo da agência** — não só repasse pra creator, mas fluxo de caixa, contas, categorias. Pora chega perto, ninguém mais.

Esses três pilares são onde o discurso de venda do Kanvas pode ser único: **"o ERP da agência, não só backoffice de pagamento"**.

## Melhorias priorizadas (ordem sugerida)

### Bloco A — Fechar gaps que o mercado considera obrigatórios

Sem esses, agência olha o sistema e fala "não substitui o que tenho hoje". Já documentei o detalhe em `gap-contratos-e-pagamentos.md`.

**A1. Contrato com assinatura digital** (3-5 semanas)
- Detalhe completo no doc de gap
- Provider sugerido: ZapSign ou Autentique
- Domínio: `Contract`, `ContractTemplate`, `ContractSignature`

**A2. Pagamento ao creator com gateway PIX** (4-6 semanas)
- Domínio: `CreatorPayment`, `CreatorBankAccount`, `CreatorInvoice`
- Gateway sugerido: Asaas (split nativo, custo baixo)
- Pagamento em lote com 1 clique (todos os concorrentes têm)

**A3. Portal do creator** (2-3 semanas)
- Login simples para creator visualizar campanhas, contratos, pagamentos
- Hoje o sistema só tem links públicos isolados (`ProposalPublicController`, `DeliverablePublicController`)
- Precisa de área autenticada do creator
- Relevante porque desbloqueia A1 (assinatura) e A2 (NF + dados bancários)

### Bloco B — Diferenciar e modernizar

**B1. Coleta de dados via "link mágico" estilo Pora** (1 semana)
- Link único enviado ao creator que coleta: dados pessoais, dados bancários, NF, foto, redes sociais
- Token expirável, não exige login
- Reduz fricção de onboarding (Pora vende isso como diferencial central)

**B2. White label completo** (2-3 semanas)
- Cores primária/secundária por tenant (já tem multi-tenant pronto)
- Logo da agência
- Domínio próprio (`agencia.cliente.com.br`)
- Customização de templates de email e contrato
- Pora vende muito isso. Tecnicamente fácil em cima do Archon multi-tenant.

**B3. Briefing estruturado e padronizado** (1-2 semanas)
- Hoje a Campaign tem campos livres
- Concorrentes têm briefing com seções definidas: objetivo, público, tom de voz, do's & don'ts, hashtags obrigatórias, links de tracking, datas-chave, exemplos de referência
- Modelo: criar `CampaignBriefing` value object dentro de `Campaign` ou agregado separado
- Plus: gerar briefing em PDF compartilhável

**B4. Calendário de postagens (módulo Operations)** (2 semanas)
- Visão calendário das `CampaignDeliverable` com datas planejadas e datas reais
- Drag-and-drop pra reagendar
- Pora e Unfy têm. Encaixa em Operations que já existe no frontend.

**B5. Aprovação de roteiro antes do conteúdo** (1-2 semanas)
- Inflr e Buzzcreators destacam isso como fluxo
- Adicionar etapa `ScriptApproval` antes da `DeliverableApproval`
- Reduz retrabalho real (creator grava, agência rejeita, creator regrava)

**B6. Chat embutido por campanha/creator** (3-4 semanas)
- Brinfer, Post2B e Buzzcreators têm
- Mensagens entre agência ↔ creator dentro da plataforma
- Histórico fica auditável (importante pra disputas)
- Tecnicamente: SignalR + agregado `Conversation` + `Message`
- Mais complexo. Avaliar se vale antes de B1-B5.

### Bloco C — Avançado / IA / Dados

Cuidado com esse bloco. É caro construir e exige base de dados de redes sociais que Kanvas não tem hoje.

**C1. Captura automática de posts/stories** (~ 4-8 semanas + custo recorrente)
- Influency, Unfy e outros têm
- Exige integração com Instagram Graph API, TikTok Business API, YouTube Data API
- Apple-to-apple impossível (Instagram limita stories drasticamente desde 2023)
- Alternativa realista: link de post + scraping leve para métricas snapshot
- Considerar **só depois** de A1, A2, A3 prontos

**C2. Banco/marketplace de creators** (~ 8-12 semanas + GTM próprio)
- Influency tem 100k creators, Unfy tem 100k, Spark e Air têm bancos consolidados
- Não dá pra competir do zero — leva anos construir
- **Recomendação: NÃO construir.** Vai virar feature ruim. Se precisar de descoberta, integrar com API de terceiro (HypeAuditor, Modash) ou parceria com player que já tem.

**C3. IA para matching/recomendação** (~ 4-6 semanas + custo OpenAI/Anthropic)
- Unfy e Air destacam isso
- Faz sentido só **depois** de ter banco de creators (própria ou via API)
- Sem dados, IA não tem o que recomendar

**C4. Monitoramento de concorrentes** (~ 4-6 semanas)
- Brinfer destaca isso
- Demanda raspagem ou API de social listening
- Baixa prioridade — não é dor central de agência hoje

### Bloco D — Higiene técnica (já mapeado na análise inicial)

Independente das features de produto, esses pontos ficam:

- D1. Remover credenciais hardcoded de `appsettings.json` (1 dia)
- D2. Implementar fila de email real (substituir `Console.WriteLine` da "Fase B") (3-5 dias)
- D3. Criar suite de testes unitários backend mínima (2-4 semanas, contínuo)
- D4. Vite build args via env (não hardcoded no Dockerfile) (1 dia)
- D5. Migration strategy Dev vs Prod (não rodar `RunMigrations` automático em prod) (2 dias)

## Roadmap sugerido (12 meses)

**Q1 (3 meses)** — Bloco D higiene + Bloco A (contratos + pagamentos + portal creator)
- Fecha o gap obrigatório
- Sistema vira "viable" pra qualquer agência olhando

**Q2 (3 meses)** — Bloco B prioritário (B1 link mágico, B2 white label, B3 briefing, B4 calendário)
- Sistema fica visualmente competitivo com Pora
- White label permite revenda

**Q3 (3 meses)** — Bloco B avançado (B5 aprovação roteiro, B6 chat) + relatórios em PDF/XLS
- Fecha gaps de fluxo de trabalho
- Sistema vira "melhor que Pora em pipeline + propostas + financeiro" mantendo paridade no resto

**Q4 (3 meses)** — Bloco C seletivo (C1 captura básica de posts via API)
- Avaliar IA só se houver base de uso real
- Não construir banco de creators do zero

## O que NÃO fazer (decisão estratégica)

1. **Não construir banco de creators.** Mercado tem 5+ players com 50k+ creators cada. Esse jogo está perdido.
2. **Não focar em descoberta.** É o mesmo jogo do banco de creators. Foco do Kanvas é **operação interna da agência**, não matchmaking.
3. **Não competir com Pora em antecipação financeira própria.** Eles têm aprovação BACEN. Você não. Se precisar antecipação, integrar com Pora ou parceiro financeiro.
4. **Não copiar Influency/Unfy em IA.** Sem base de dados, IA é teatro. Construir IA só depois de ter dados próprios.

## Resumo executivo

O Kanvas está num **espaço pouco contestado**: ERP operacional da agência (pipeline → proposta → campanha → entrega → financeiro) num único produto.

Os concorrentes ganham em três frentes onde o Kanvas não tem nada: **contrato digital, pagamento ao creator, portal do creator**. Fechar isso (Bloco A) coloca o Kanvas em paridade.

Ganham também em features visuais e de fluxo: **link mágico, white label, briefing estruturado, calendário, aprovação de roteiro, chat**. Todas tecnicamente acessíveis (Bloco B).

Perdem na essência onde o Kanvas é único: **CRM comercial + propostas rastreáveis + financeiro completo + automações**. Esse é o discurso de posicionamento.

Próximo passo concreto: escolher entre começar pelo Bloco D (higiene técnica, 1-2 semanas) ou pelo Bloco A1 (contratos, 3-5 semanas).
